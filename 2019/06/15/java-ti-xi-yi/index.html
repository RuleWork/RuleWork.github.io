Java体系(一)
Java体系

阅读时长:   8 分

前言：很长时间没写博客了，目前大三，通过这段时间的学习，对技术有了宏观上的视野，对Java的生态体系也有了较深的理解，所以停止了动笔主要是因为发觉之前自己的知识体系有很大的漏洞，IT所有技术的学习就好比数据库建索引的过程，如果没有清晰认知到整个技术的骨架，难免会步入迷茫的困境，所以这段时间主要用来学习各种知识，构建自己的Java体系，这个系列的博客大概会更多期，第一篇我先从湿货谈起。

一.技术的发展趋势

IT技术的更新真的是非常之快，近年来也可以明显的发现目前程序员岗位的门槛已显著提高，所以我认为技术视野和发展趋势对于任何一个程序员都是至关重要的，无论你是刚打算入行还是已经工作几年，如果方向不对，势必会大浪淘沙。

就拿Java来说，现在早已不是哪个学学JavaSE懂点JavaWeb和框架就能找到工作的时期了，这里我随便挑一份标准的上万月薪的Java招聘简历举例：

可以明显的看出目前整个市场对技术深度和广度都提出了更高的要求，这也是必然的，目前互联网行业已经进入了洗牌阶段，人口红利已趋近吃完，刚过去的互联网寒冬也给很多人敲醒了警钟。

但我认为市场仍有很多人和媒体在过度贩卖焦虑，说IT已经濒临饱和，亦或是江河日下，我认为这是无稽之谈。就目前的市场来看，虽然经过996的热潮揭示出IT行业工作时长与时薪不成正比的问题，但IT仍然是高薪的代名词，这是毋庸置疑的。目前来看，移动互联网的红利显然已经吃完了，低阶的安卓和IOS人员已经需求量明显下降，小程序和大前端的概念和趋势也会愈加蚕食移动端的市场。不过打铁还需自身硬，如果有能力迈入高阶水平，还是可以有很多作为的，短期来看，能够再次引燃科技市场，带来新的变革的技术我认为是5G，区块链和人工智能。

如果你逛过知乎，你会发现不知多少大V都在宣扬着转IT的政治正确言论，经常会有人这样的提问：、
多少多少岁，该不该转IT?
什么什么行业，该不该转IT?
什么什么专业，该不该转IT?

很多人就会产生错觉，认为IT简单，钱多，有前途。这明显是认知误区，可以明显的发现其他行业有利可图都生怕你来占坑，IT却好像是明显有利可图，巴不得你来占坑，这真是一个有趣的现象。

说到底，IT绝对不是一个低门槛的行业，虽然IT开源界有着独一无二的学习条件，但热爱编程，有技术热情的人其实很少，冲钱来的话，我断定你不会走太远，所以针对这些问题我只有一个评判标准，就是你是否有技术热情和对编程的敏锐度？，如果你看见代码就头疼，盯见bug就头大，我劝你还是不要趟这趟浑水了，适合自己的，永远是最重要的。

二.你应该选择什么技术

还是那句话，适合自己的，永远是最好的，只要是市场上有需求的你都可以选择，但是如果你实在不知道具体哪个技术适合你，那么我还是优先推荐Java，当然前端发展也很不错。

如果有人告诉你，语言不重要，重要的是思维和基础，你就当他说废话好了，思维和基础那是素质，语言那是你吃饭的东西，且我们在说一门语言的时候仅仅是在讨论一门语言吗？显然不是，我们更看重的是这门语言背后的整个生态体系，社区活跃度，发展前景等等，当然选择一门语言一定不是说别的语言就一点也不接触，而是说这门语言是你的主要语言，你需要长久坚持，并搞深搞精！

至于为何推荐Java，原因很简单，我们来看招聘岗位的需求就可以了：

岗位的需求体现着一门技术的热门程度和找工作的难易程度，显然选择Java是没有问题的。

且从学习曲线的角度，Java要比C++平滑太多，下面简要介绍一下这些主流技术的应用范围和发展前景。

前端：目前很火，跨平台开发的Futter，服务端的NodeJS，桌面端的electron，以及各种成熟的框架，应用范围很广，包括小程序，网页等所有设备的前端都可以涉猎。

C++: C++主要用来做高性能服务器端的开发，游戏引擎开发，windows桌面端QT开发，嵌入式等对性能实时性要求极高等场景，C++以陡峭和学习曲线和较高的开发成本著称。

算法：门槛较高，对学历要求较高，对数学功底要求较高，目前主要以机器学习，图像处理，NLP，数据挖掘为主，主要应用领域有广告，推荐，搜索等。

Python：典型的胶水语言，应用范围很广，库非常多，可以用来做算法，后端，数据分析，爬虫，自动化运维等。

C#：不错的语言，但生态环境不好，主要用来做桌面的winform开发，unity3D开发，.net开发等。

PHP :构建小型网站不错的选择，学习成本低，库多，开发速度快。

IOS/安卓：主要做两个系统的各种app开发。

Go：Docker/k8s的火爆，让go在云容器市场有很大的发挥空间，典型的就是七牛云，除此之外在后端开发也日渐上升，B站的后端就全部采用了go，在区块链领域也发展不错，go的生态也很好。

那么Java能做什么呢？
坦白讲Java几乎什么都能做，各方面都很均衡，其安全性，健壮性，跨平台让Java可以开发大型的复杂系统，但大范围的应用领域是Web系统的后端开发，大数据开发，也可以深入学习Netty和多线程去做服务器开发，还有一些冷门领域例如桌面GUI的swing/javaFx开发，游戏开发等，很多人都以为Java不能做游戏前端的开发，其实不然，最近在github上发现了一个Java游戏开发引擎，非常不错，如果想自己做做游戏的，可以了解一下。

链接：https://github.com/jMonkeyEngine/jmonkeyengine

如果你还想从语言特性，语言设计等角度深层次的角度深入了解为何选择Java，建议你看这篇博客：

http://www.yinwang.org/blog-cn/2016/01/18/java

随着Java5的JUC，Java8的Lamada以及Java11提出的ZGC，很多人诟病的Java并发支持不够，语法臃肿代码量大，垃圾回收效率不够高等各种问题已经被解决，而且红帽被IBM收购，OpenJDK又被红帽接盘，Java的发展从未停滞，所以从任何角度来讲Java仍然是非常好的工业级语言。

三.深度还是广度？

深度和广度都很重要，需要自己权衡，最好是有一门很深入的，辅以其他均衡的，呈丁字型发展。

广度体现在对多种技术的涉猎上，具体拿Java来说，除了必备的框架之外，分布式，消息队列，缓存，微服务的要求已经越来越普遍。

深度体现在某一领域的深挖上，无论是技术还是业务的深挖都非常重要，同时体现在对基础，算法，底层的掌握程度上，比如JVM，数据结构，组成原理等。

当然这些都是要在工作中逐步积累，不断学习，才能厚积薄发。

四.总结

这篇博文主要是自己对技术视野的一个整体看法，个人认为还是非常重要的，之后我会主要针对Java体系，写出详细的干货，这些干货有些是自己摸索的产物，有些是各种前辈的总结，包括学习路线，学习方法，学习途径等，一方面是对自己构建知识体系的一种总结，另一方面也更希望能给更多人带来一定的帮助，我也会不断修改自己的措辞，保证语句尽可能的通俗易懂，简洁清晰。
